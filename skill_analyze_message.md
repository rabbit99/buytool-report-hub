# Skill: 新聞記者式訊息分析

## 目的
將原始 LINE 群組聊天訊息轉化為**新聞記者風格的客觀陳述**，突出事實、數據和共識。

## 輸入格式
```
訊息群組: [群組名稱]
分類: [主題分類]
子分類: [子主題]
提及人數: [N 人]
訊息列表:
- [訊息1]
- [訊息2]
...
```

## 分析要求

### 1. 事實提取
- 抽取具體事實、數據、操作步驟
- 避免個人觀感，改用「據回報」、「多人表示」、「有使用者提出」等客觀用語

### 2. 共識識別
- 若多人提及相同觀點 → 「多數參與者表示」
- 若少數人提及 → 「部分用戶提到」或「有人反應」
- 若存在對立意見 → 「意見分歧，部分人認為...，而另有人指出...」

### 3. 數據與細節
- 優先突出具體數字、時間、條件
- 例如：「某使用者測試在 24 小時半自動掛野外，未出現被鎖狀況」
- 量化人數：幾人→「少數」，5-10人→「部分」，10+人→「多數」

### 4. 語氣要求
- 客觀、平衡、新聞體
- 避免絕對化詞彙：不用「一定」、「絕對」，改用「多數」、「未見報告」
- 合理懷疑的表述：「尚無確認」「有待驗證」

## 輸出格式

每條訊息分析輸出 **一個連貫段落**：

```
【標題】[用一句話概括議題]
【分析】[客觀新聞體段落，1-3句，包含人數、具體事實、相關條件]
【補充】[可選] 若有對立意見或待驗證部分
```

## 範例

### 輸入
```
訊息群組: 特工交流群
分類: 掛機攻略
子分類: 地點安全性
提及人數: 3 人
訊息列表:
- 我掛野外 24小時 半自動 也沒有被鎖啊
- 野外確實比較安全，我掛了一星期都沒事
- 野外黑怪多，掛久了容易掉線
```

### 輸出
```
【標題】野外掛機安全性評估
【分析】多數參與者測試在野外環境進行 24 小時至一週的半自動或全自動掛機，未出現被鎖狀況。惟一名用戶反應野外黑怪密集可能導致掉線風險，安全性與環境因素相關。
【補充】建議搭配防掉線對策；野外作為相對安全的掛點存在共識。
```

## 邊界情況

- **純提問**（如「怎樣不會被鎖？」）→ 分類為「未答詢問」，不輸出分析
- **廣告或無關內容** → 跳過，標記「已過濾」
- **個人經驗，無可驗證性** → 記為「個人回報」，但仍可納入趨勢
- **強烈主觀意見** → 記為「觀點」而非事實

## 質量檢查清單

- [ ] 避免使用「我認為」、「應該」等主觀詞彙
- [ ] 人數量化（多少人 → 多數/少數/部分）
- [ ] 具體數據優先（時間、數值、條件）
- [ ] 平衡呈現不同聲音
- [ ] 合理標註不確定性（如「尚未確認」、「有待驗證」）
